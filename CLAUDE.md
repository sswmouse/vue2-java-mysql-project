# 项目规范

## 代码规范
1. 使用 ESLint 进行代码规范，修改后要检查项目的 ESLint 报错并进行修复
2. 提供的代码要进行注释：方法注释、文件头注释、变量注释
3. **最小修改原则**：不要影响历史功能

## 测试规范
1. 测试截图统一保存到项目根目录的 `test-screenshots/` 文件夹
2. 截图命名格式：`YYYY-MM-DD-功能描述.png` 或 `YYYY-MM-DD-页面-状态.png`
3. 功能测试完成后要及时删除测试截图，保持文件夹整洁

## 工作流程
1. 任务要进行拆分
2. 修改完成后要测试功能是否正常，后端改动后要重新编译 + 启动
3. 每次更新新内容需要修改更新 md 文件
4. 遇到不明确的问题要咨询我

1. 使用中文沟通
2. 保持回答简洁
